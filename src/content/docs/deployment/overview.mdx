## Deployment Flow

Deploying the ECS Express Mode demo is a two-command workflow: `terraform init` to initialize the working directory and download providers, then `terraform apply` to provision the entire stack. The <Tooltip term="Express_Gateway_Service">Express Gateway Service</Tooltip> resource handles most of the infrastructure wiring automatically.

<Aside type="note">
  All resources are created in a single apply. There is no multi-step deployment
  or manual ALB configuration required — Express Mode provisions the load
  balancer, target group, listener, and auto scaling as part of the service
  creation.
</Aside>

## Verifying the Deployment

After `terraform apply` succeeds, verify the deployment is healthy by checking three things:

1. **Terraform outputs** — retrieve the <Tooltip term="Application_URL">Application URL</Tooltip> and ALB DNS name from `terraform output`
2. **Service status** — confirm desired count equals running count in the ECS console or via `aws ecs describe-services`
3. **Health endpoint** — curl the health check path and confirm an HTTP 200 response

<Aside type="tip">
  Use the Application URL (<code>*.ecs.&lt;region&gt;.on.aws</code>) for all
  application access rather than the raw ALB DNS name. The Application URL is
  the Express-provisioned endpoint with Host header routing configured.
</Aside>
