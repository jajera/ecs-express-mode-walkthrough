ECS Express Mode — Core Concepts Description
Express Gateway Service: Single Terraform resource (aws_ecs_express_gateway_service) that provisions the entire HTTP/HTTPS stack.
Managed ALB: ECS provisions a Shared Application Load Balancer — no separate aws_lb, target group, or listener resources.
Health Checks: ALB target group health checks configured via health_check_path on the express gateway service.
Auto Scaling: Built-in target tracking via scaling_target — no separate aws_appautoscaling_target.
Application URL: Unique HTTPS endpoint on *.ecs.<region>.on.aws for all normal application traffic.
Classic ECS Alternative: Requires separate aws_lb, aws_lb_target_group, aws_lb_listener, aws_ecs_service, and aws_appautoscaling_target wired together manually.
